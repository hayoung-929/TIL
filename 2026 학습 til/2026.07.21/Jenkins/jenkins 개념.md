# Jenkins

## 1. Jenkins란?

**내 서버에 직접 설치해서 돌리는 "자동화 로봇(집사)".**
코드가 올라오면 → 빌드 → 테스트 → 배포를 사람 대신 자동으로 반복 실행해준다.

- **Java로 만든 웹 서버** → `localhost:8080`으로 접속해서 화면을 봄
- 작업(Job)을 안 만들면 → 그냥 **명령을 기다리는 빈 대기 상태**
- Job을 등록하는 순간부터 → 대기 중이던 자동화 엔진이 실제로 일을 시작

---

## 2. 자동화의 이점

사람이 직접 하면: 코드 합칠 때마다 빌드·테스트·배포를 손으로 → 느리고, 실수 가능

Jenkins로 자동화하면:

- **반복 제거** — 한 번 설정하면 앞으로 영원히 자동
- **실수 방지** — 사람이 실수할 일이 없음
- **빠른 피드백** — 실패하면 즉시 알림

---

## 3. 작동 방식 (4단계)

```
① Trigger  →  ② Controller가 일 시킴  →  ③ 순서대로 실행  →  ④ 결과 알림
```

1. **Trigger** — 언제 시작할까?
  - 코드 push / 정해진 시각 / 사람이 직접 버튼 클릭
2. **Controller(두뇌)** — 정의해둔 Job을 꺼내서 Agent(일꾼)에게 시킨다
3. **순서대로 실행(Stage)** — 코드 가져오기 → 빌드 → 테스트 → 배포
  - **앞 단계가 실패하면 거기서 멈춤** (테스트 깨지면 배포 안 함 = 안전장치)
4. **결과 알림(Feedback)** — 성공/실패, 이메일·Slack 알림



### 주요 부품

- **Controller** : 지시하는 두뇌
- **Agent** : 실제 빌드·테스트를 하는 일꾼 (소규모면 Controller가 겸함)
- **Plugin** : 기능을 붙이는 부품 (Git 연동, Docker, Slack 등)

## 4. Jenkins vs GitHub Actions


| 항목          | Jenkins               | GitHub Actions  |
| ----------- | --------------------- | --------------- |
| 어디서 작용하나?   | **내가 설치한 내 서버**       | **GitHub 클라우드** |
| 설치          | 필요 (Java + 프로그램)      | 불필요             |
| 설정 방법       | 웹 화면 클릭 / Jenkinsfile | YAML 파일         |
| 관리 부담       | 사용자가 함                | GitHub이 대신 함    |
| GitHub 필수인가 | 독립적                   | 예               |
| 비용          | 프로그램 무료, 서버는 내 부담     | 무료 한도 + 초과분 과금  |




### 한줄

- **Jenkins** : 다 커스터마이징 가능하지만 관리는 내 몫
- **GitHub Actions** = 편하지만 GitHub에서만 사용



### 그럼 왜 Jenkins를 배우나?

1. 코드가 GitHub에 없는 회사가 많음
2. 보안·규제 산업은 외부 클라우드 빌드 금지 (금융·의료·공공)
3. 아주 복잡한 자동화는 플러그인 1,800개의 Jenkins가 유연
4. 이미 Jenkins로 구축된 회사가 많음 → **취업 시장 수요 여전히 큼**

## 5. 용어 5개


| 용어              | 뜻                                    |
| --------------- | ------------------------------------ |
| **CI** (지속적 통합) | 코드를 자주 합치고 합칠 때마다 자동 빌드·테스트          |
| **CD** (지속적 배포) | 테스트 통과한 코드를 자동으로 서버에 배포              |
| **Pipeline**    | 빌드→테스트→배포 작업 순서를 정의한 흐름              |
| **Job / Item**  | Jenkins가 수행하는 작업 하나 (예: `hello`)     |
| **Plugin**      | 기능을 붙이는 부품 (Git, Docker, Slack 연동 등) |
